### <a id="image"></a> **[image] 显示图像**



属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|storage|设定要显示的图像文件。背景图请放在data/bgimage文件夹里，普通图片放进data/fgimage文件夹里|
|layer|设定要显示图像的图层。base为背景层，单个数字为前景层|base|
|page|设定要显示的图层在背景还是前景。如果未设定该属性，默认前景|fore|
|visible|设定可见性。该属性也会改变图像所在的图层可见性，这意味着不必使用[layopt]标签|
|left|设定图像距离左侧边缘的距离|
|top|设定图像距离顶部边缘的距离|
|x|设定图像的x轴位置，该属性优先级高于left|
|y|设定图像的y轴位置，该属性优先级高于top|
|width|设定图像的宽度|
|height|设定图像的高度|
|folder|设定要显示图像所在的文件夹|fgimage|
|name|该变量给[anim]使用，用于让[anim]为其添加动画。同时，也可在HTML中作为class使用|
|time|设定显示图像所需的时间，该显示动画被固定为fadeIn。如果未设定则直接显示|
|wait|设定是否等待显示图像动画完成|true|
|zindex|设定图像的层级顺序，也叫堆叠顺序。数字越大，该图像越靠前显示|1|
|depth|如果zindex的数值相同，请依靠该属性确定不同图像应该显示在前景（`front`）还是背景（`back`）|front|
|reflect|设定图像是否水平翻转，如果要翻转请设定为“`true`”|
|pos|该属性用于确定图像的位置。<br />可用的关键字有“`left`”（左边缘）、“`left_center`”（左对齐）、“`center`”（居中）、“`right_center`”（右对齐）和“`right`”（右边缘）。每个关键词的具体坐标在Config.tjs中有明确定义，并且可以自由编辑。每个关键词还有简写形式，例如`l`、`lc`、`c`、`rc`和`r`，其功能相同。<br />如果设定此属性，则`left`属性将被忽略。<br />如果该层是`base`，则无需指定此属性|
|animimg|如果使用的图像文件是动图（例如gif和apng），请设定此属性为`true`|

代码结构及示例：
```
[image storage="example.png" x="10" y="10" layer="0" page="fore" visible="false" left="1" top="1" width="1280" height="720" folder="fgimage" name="example" time="3000" wait="true" zindex="1" depth="front" reflect="true" pos="left" animimg="true" ]
```
  
### <a id="freeimage"></a> **[freeimage] 清除图层上的所有图片和其他内容**

该标签用于删除指定图层上的所有图片和其他内容。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|layer|设定要清除的图层||〇|
|page|设定要清除的是图层前景上的图片还是背景|fore|
|time|设定图片消失所需的时间。设置该属性后，图片会以淡出动画的方式消失|
|wait|设定是否等待图片消失完成。若设置为“true”，则游戏会暂停等待图片消失|false|

代码结构及示例：
```
[freeimage layer="0" page="fore" time="1000" wait="true"]
```

### <a id="backlay"></a> **[backlay] 将图层的前景内容复制到其背景**

该标签用于将图层的前景内容复制到其背景。

此标签与[trans]标签配合使用。首先，使用[backlay]标签将前景的图像复制到背景，在背景对图像进行修改，然后使用[trans]标签将背景的图像带到前景。这样就不用担心游戏屏幕上会显示图像的修改过程。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|layer|设定目标图层。使用“base”表示背景图层，使用更高的整数表示前景图层，使用“message0”或“message1”表示消息图层。如果仅指定“message”，则当前由 [current] 标签指定的消息图层将成为目标。如果省略，则所有图层的信息都将复制到背面页面|

代码结构及示例：
```
[backlay layer="0"]
```

### <a id="layopt"></a> **[layopt] 图层属性设定**

该标签用于对图层进行设定。例如图层的是否可见。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|layer|设定要更改属性的图层||〇|
|page|设定是要对前景操作，还是背景操作。但若仅在“layer”属性中指定“message”，而该属性又被省略，则将选择当前所选定的页面|fore|
|visible|设定图层是否可见。true为可见，false为不可见|
|left|设定图层的x轴位置。如果你要对消息窗口操作，请使用[position]而不是这个|
|top|设定图层的y轴位置。如果你要对消息窗口操作，请使用[position]而不是这个|
|opacity|设定图层的不透明度。取值范围为0-255，255为不透明|

代码结构及示例：
```
[layopt layer="0" page="fore" visible="true" left="100" top="100" opacity="255"]
```
  
### <a id="locate"></a> **[locate] 指定显示位置**

该标签用于调整图片按钮的位置。

比较形象的解释是，请把它想象成一个鼠标光标，在这个光标的位置放置图片。

### <a id="trans"></a> **[trans] 过渡**

对指定的图层实行过渡效果。

该标签会将背景的内容移动到前景，以实现指定图层的过渡效果。完成过渡后，前景内容（图像、位置、大小、可见性等）将与背景内容相同。请注意在转换过程中不要更改图层属性。

注意，此时背景内容变为前景，前景内容变为背景。如果要转换回去，请执行相同的代码块。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|layer|设定目标图层。使用“base”表示背景图层，使用更高的整数表示前景图层，使用“message0”或“message1”表示消息图层。如果仅指定“message”，则当前由 [current] 标签指定的消息图层将成为目标图层|base|〇|
|time|设定过渡动画所需的时间|1500|〇|
|method|设定过渡动画类型。根据版本的不同，有以下参数分类：<br />【V450以后】`fadeIn`、`fadeInDown`、`fadeInLeft`、`fadeInRight`、`fadeInUp`、`lightSpeedIn`、`rotateIn`、`rotateInDownLeft`、`rotateInDownRight`、`rotateInUpLeft`、`rotateInUpRight`、`zoomIn`、`zoomInDown`、`zoomInLeft`、`zoomInRight`、`zoomInUp`、`slideInDown`、`slideInLeft`、`slideInRight`、`slideInUp`、`bounceIn`、`bounceInDown`、`bounceInLeft`、`bounceInRight`、`bounceInUp`、`rollIn`、`vanishIn`、`puffIn` <br />【V450以前】`crossfade`、`explode`、`slide`、`blind`、`bounce`、`clip`、`drop`、`fold`、`puff`、`scale`、`shakesize`|fadeIn|
|children|[已弃用]设定是否仅应用指定的层过渡效果|false|

代码结构及示例：
```
[trans layer="base" time="1500" method="fadeIn" ]
```
  
### <a id="bg"></a> **[bg] 切换背景**

该标签用于显示或切换背景。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|storage|设定要作为背景的图像文件。该文件请放置在data/bgimage文件夹里||〇|
|time|设定切换背景所需的时间|3000|
|wait|设定是否等待背景切换完成。如果属性为“false”，那么引擎读取完此标签后会立刻读取下一个标签|true|
|cross|设定切换前和切换后的图像是否同时进行过渡，即通过变为透明状态来实现相互转换。若为“false”，则新的背景会覆盖在旧背景之上，但不会将其移除。这在CG界面非常有用|false|
|position|设定背景图像位置。该属性的目的是，保持图像原本比例而不是填满整个屏幕，因此需要确定图像的相对位置。该属性为空时，图像将填满整个屏幕。可用的参数有`left`（左对齐）、`center`（居中）、`right`（右对齐）、`top`（上对齐）、`bottom`（下对齐）|
|method|设定切换动画类型。根据版本的不同，有以下参数分类：<br />【V450以后】`fadeIn`、`fadeInDown`、`fadeInLeft`、`fadeInRight`、`fadeInUp`、`lightSpeedIn`、`rotateIn`、`rotateInDownLeft`、`rotateInDownRight`、`rotateInUpLeft`、`rotateInUpRight`、`zoomIn`、`zoomInDown`、`zoomInLeft`、`zoomInRight`、`zoomInUp`、`slideInDown`、`slideInLeft`、`slideInRight`、`slideInUp`、`bounceIn`、`bounceInDown`、`bounceInLeft`、`bounceInRight`、`bounceInUp`、`rollIn`、`vanishIn`、`puffIn` <br />【V450以前】`crossfade`、`explode`、`slide`、`blind`、`bounce`、`clip`、`drop`、`fold`、`puff`、`scale`、`shakesize`|fadeIn|

代码结构及示例：
```
[bg storage="example.jpg" time="3000" wait="true" cross="false" position="center" method="fadeIn" ]
```
  
### <a id="bg2"></a> **[bg2] 切换背景**

该标签用于显示或切换背景。

该标签与[bg]标签不同，[bg]标签受制于page属性的影响，也就是说，由[bg]标签定义的背景，可以通过[image]标签为base图层的背景添加图像。本标签则禁止这种操作。

这就意味着，该标签只能修改前景内容而无法使用[trans]过渡。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|name|该变量给[anim]使用，用于让[anim]为其添加动画。同时，也可在HTML中作为class使用|
|storage|设定要作为背景的图像文件。该文件请放置在data/bgimage文件夹里||〇|
|left|设定图像的x轴位置|0|
|top|设定图像的y轴位置|0|
|width|设定图像的宽度。如果未设定此属性则填满整个屏幕|
|height|设定图像的高度。如果未设定此属性则填满整个屏幕|
|time|设定切换背景所需的时间|3000|
|wait|设定是否等待背景切换完成。如果属性为“false”，那么引擎读取完此标签后会立刻读取下一个标签|true|
|cross|设定切换前和切换后的图像是否同时进行过渡，即通过变为透明状态来实现相互转换。若为“false”，则新的背景会覆盖在旧背景之上，但不会将其移除。这在CG界面非常有用|false|
|method|设定切换动画类型。根据版本的不同，有以下参数分类：<br />【V450以后】`fadeIn`、`fadeInDown`、`fadeInLeft`、`fadeInRight`、`fadeInUp`、`lightSpeedIn`、`rotateIn`、`rotateInDownLeft`、`rotateInDownRight`、`rotateInUpLeft`、`rotateInUpRight`、`zoomIn`、`zoomInDown`、`zoomInLeft`、`zoomInRight`、`zoomInUp`、`slideInDown`、`slideInLeft`、`slideInRight`、`slideInUp`、`bounceIn`、`bounceInDown`、`bounceInLeft`、`bounceInRight`、`bounceInUp`、`rollIn`、`vanishIn`、`puffIn` <br />【V450以前】`crossfade`、`explode`、`slide`、`blind`、`bounce`、`clip`、`drop`、`fold`、`puff`、`scale`、`shakesize`|fadeIn|

代码结构及示例：
```
[bg2 name="example" storage="example.jpg" left="10" top="10" width="1280" height="720" time="1000" wait="true" cross="false" method="fadeIn" ]
```
