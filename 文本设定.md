### <a id="config_record_label"></a> **[config_record_label] 已读文本状态设置**

允许更改已读文本的状态设置。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|color|改变已读文本的颜色，以0xRRGGBB格式|
|skip|设定是否允许玩家跳过未读文本。当该值为false时将禁止玩家在未读文本处快进|

代码结构及示例：
```
[config_record_label color="0xffffff" skip="false" ]
```
  
### <a id="position"></a> **[position] 消息窗口的属性改变**



属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|layer|设定要显示文本的消息层|message0|
|page|设定要在消息层显示文本的是前景还是背景|fore|
|left|设定消息窗口的x轴位置|
|top|设定消息窗口的y轴位置|
|width|设定消息窗口的宽度|
|height|设定消息窗口的高度|
|frame|指定要作为消息窗口背景显示的图片。该图片的尺寸应与宽度和高度属性相匹配。使用margin属性来调整实际的消息显示区域。将值设为“none”可恢复到默认框架设置|none|
|color|设定消息窗口的背景颜色，以0xRRGGBB格式|
|border_color|设定消息窗口的边框颜色，以0xRRGGBB格式|
|border_size|设定消息窗口的边框大小。值为0时不显示边框|0|
|opacity|设定消息窗口的透明度，数值范围为0到255|
|marginl|设定显示文本区域的窗口的左侧外边距|
|margint|设定显示文本区域的窗口的顶部外边距|
|marginr|设定显示文本区域的窗口的右侧外边距|
|marginb|设定显示文本区域的窗口的底部外边距|
|margin|此属性为marginl、margint、marginr、marginb的合并设定，格式为“top,left,right,bottom”|
|radius|设定消息窗口的圆角。有如下参考值：10（略微圆润型）；30（中等圆润型）；100（高度圆润型）|
|vertical|设定是否启用竖排文字。true为竖排，false为横排|
|visible|设定消息窗口的可见性|
|gradient|V515版本之后，设定背景使用的渐变效果，空值表示不启用，该值使用CSS的linear-gradient()函数|

代码结构及示例：
```
[position layer="message0" page="fore" left="10" top="10" width="100" height="100" frame="example.jpg" color="0xffffff" border_color="0xffffff" border_size="0" opacity="255" marginl="10" margint="10" marginr="10" marginb="10" radius="30" vertical="false" visible="true" gradient="linear-gradient(45deg, red 0 50%, blue 50% 100%)" ]
```
```
[position layer="message0" page="fore" left="10" top="10" width="100" height="100" frame="example.jpg" color="0xffffff" border_color="0xffffff" border_size="0" opacity="255" margin="10,10,10,10" radius="30" vertical="false" visible="true" gradient="linear-gradient(45deg, red 0 50%, blue 50% 100%)" ]
```

### <a id="font"></a> **[font] 字体样式的变更**

该标签将修改文本里的字体样式。

如果某个参数未被指定，则其设置将保持不变。指定默认值会将值重置为默认值（这在 Config.tjs 中设置或通过 [deffont] 标签指定）。

使用此标签更改的样式在执行诸如 [resetfont]、[ct]、[cm] 和 [er] 等标签时会恢复为默认值。若要永久更改默认值，请编辑 Config.tjs 或使用 [deffont] 标签。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|size|设定字体大小|
|color|设定字体颜色，以0xRRGGBB格式|
|bold|设定字体是否加粗|
|italic|设定字体是否斜体|
|face|设定字体样式。样式在tyrano/css/font.css中定义|
|edge|设定文本边框，使用0xRRGGBB格式设定颜色，空值表示不启用，在V515版本之后，按照2px 0xffffff的格式可以设定边框粗细和颜色，粗细由像素px设定，颜色以0xRRGGBB格式设定
|edge_method|选择文本边框渲染方式。可用选项：阴影（shadow）或滤镜（filter）|
|shadow|设定是否使用阴影，使用0xRRGGBB格式设定颜色，空值表示不启用|
|effect|是否给字体添加文本动画效果，设定为none时不启用动画。可用的值为fadeIn, fadeInDown, fadeInLeft, fadeInRight, fadeInUp, rotateIn, zoomIn, slideIn, bounceIn, vanishIn, puffIn, rollIn|false|
|effect_speed|设定文本动画的显示时间|
|gradient|V515版本之后，设定文本使用的渐变效果，空值表示不启用，该值使用CSS的linear-gradient()函数|

代码结构及示例：
```
[font size="10" color="0xffffff" bold="true" italic="true" face="example" edge="2px 0xffffff" edge_method="filter" shadow="0xffff00" effect="fadeIn" effect_speed="1000" gradient="linear-gradient(45deg, red 0 50%, blue 50% 100%)"]
```

### <a id="message_config"></a> **[message_config] 消息层设置**

|变量名|解释|默认值|必需|
|-|-|-|-|
|ch_speed_in_click|在文本显示过程中，点击后每个字符显示的速度。该属性可用于设定快速显示全部文本。如果设置为默认值，它将保持点击前的字符显示速度不变|
|effect_speed_in_click|在文本显示期间点击后文本动画效果的播放速度。如果设置为默认值，它将保持点击前的字符显示速度不变|

|speech_bracket_float|设定是否启用括号外侧悬浮效果。这种效果将使括号脱离文本对齐，让换行后的文本与第一行文本的字保持对齐|
|speech_margin_left|设定文本整体与括号之间的间隔。该属性需要speech_bracket_float=true|
|kerning|设定是否启用字距自动调整功能。效果的显现可能会因字体、默认字母间距以及玩家所使用的浏览器而有所不同。此设置会修改“CSS”中的“font-feature-settings”属性|
|add_word_nobreak|添加不应因换行而中断的单词。请用逗号分隔多个单词进行指定。该属性用于防止出现因换行导致英文单词不完整，请用逗号分隔需要设定的多个单词|
|remove_word_nobreak|去除禁止换行中断单词列表中添加的单词。请用逗号分隔需要设定的多个单词|
|line_spacing|设定文本行距，数值以像素为单位|
|letter_spacing|设定文本字距，数值以像素为单位|
|control_line_break|设定是否启用规范化换行。启用后该功能将会强制阻止句尾标点符号在每行文本的开头，无论是描边还是其他文本效果。|
|control_line_break_chars|设定用于规范化换行的标点符号。默认列表中有、。）」』】,.)]|

代码结构及示例：

### <a id="delay"></a> **[delay] 文字显示速度设置**

该标签用于改变显示文本所需的时间。

若要无需等待即可显示文本，请使用[nowait]标签。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|speed|设定文本显示所需的时间。较小的数值会使文本显示速度加快。所指定的数值被理解为下一个字符出现之前的时间（以毫秒为单位）。例如，设置为 1000 将会使每个字符每秒出现一次。|

代码结构及示例：
```
[delay speed="1000" ]
```
  
### <a id="resetdelay"></a> **[resetdelay] 将文字显示速度恢复为默认状态**

重置文本的显示速度（重置显示文本的所需时间）

代码结构及示例：
```
[delay speed="1000" ]
这是一个文本。
[resetdelay]
```
  
### <a id="configdelay"></a> **[configdelay] 默认文字显示速度设置**

指定默认的文本显示速度。

这决定了在使用“[resetdelay]”标签时文本显示恢复的速度。若要为整个游戏（例如在配置界面）更改文本速度，请使用此标签。

注意：如果只是想出于效果目的临时更改文本速度，请使用[delay]标签，而非[configdelay]标签。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|speed|设定文本显示所需的时间。较小的数值会使文本显示速度加快。所指定的数值被理解为下一个字符出现之前的时间（以毫秒为单位）。例如，设置为 1000 将会使每个字符每秒出现一次。|

代码结构及示例：
```
[configdelay speed="1000" ]
```
  
### <a id="nowait"></a> **[nowait] 文字瞬间显示模式开启**

即时文本显示模式。在该模式下，所有文本都会立即显示出来，无需像通常那样经过逐字符的渲染过程。

代码结构及示例：
```
[nowait]
```
  
### <a id="endnowait"></a> **[endnowait] 文字瞬间显示模式关闭**

支持即时文本显示模式结束标志。引擎读取到此标签后，后续的文本显示将恢复正常。

代码结构及示例：
```
[nowait]这是一段文本[endnowait]
```

### <a id="position_filter"></a> **[position_filter] 消息窗口背景的过滤效果**

在消息窗口背景应用滤镜效果。这可用于为消息窗口营造出磨砂玻璃的效果。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|layer|设定要应用滤镜效果的消息层|message0|
|page|设定要效果应用图层前景还是背景|fore|
|remove|设定是否移除滤镜效果|false|
|grayscale|设定滤镜效果的灰度，数值范围在0到100|0|
|sepia|设定滤镜效果的棕褐色色调，数值范围在0到100|0|
|saturate|设定滤镜效果的颜色饱和度，数值范围在0到100|100|
|hue|设定滤镜效果的色调，，数值范围在0到360|0|
|invert|设定滤镜效果色调的反转程度，数值范围在0到100|0|
|opacity|设定滤镜效果的透明度，数值范围在0到100|100|
|brightness|设定滤镜效果的亮度。为0会将画面完全变为黑色，而数值超过100时则会增强亮度|0|
|contrast|设定滤镜效果的对比度，数值范围在0到100|100|
|blur|设定滤镜效果的模糊度，数值越大越模糊，最小为0|0|

代码结构及示例：
```
[position_filter layer="message0" page="fore" remove="false" grayscale="0" sepia="0" saturate="100" hue="0" invert="0" opacity="100" brightness="0" contrast="100" blur="0" ]
```

### <a id="pushlog"></a> **[pushlog] 在历史记录中添加文本**

允许在backlog中添加自定义文本。例如，使用 [mtext] 显示的文本默认情况下不会被记录在backlog中。然而，通过将 [pushlog] 与 [mtext] 结合使用，您可以将动态文本内容记录在backlog中。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|text|设定要添加到历史记录中的文本内容||〇|
|join|决定是否将新输入的内容与之前的文本合并|false|

代码结构及示例：
```
[pushlog text="这是一个自定义的历史记录条目" join="false"]
```
