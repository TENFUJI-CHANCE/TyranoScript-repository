### <a id="s"></a> **[s] 游戏停止**

该标签用于在游戏中设置一个停止点。当玩家到达这个点时，游戏将暂停，等待玩家的输入来继续游戏。

换句话说，当玩家到达 [s] 点时，就不能再通过点击或轻触来推进剧情了，所以您必须为玩家提供一种跳过该剧情的方式。

建议在使用[button][glink][link]等标签时，将它们放在[s]标签之前，以便玩家可以通过点击这些标签来继续游戏,并且阻止后续的剧本导致游戏不流畅。

代码结构及示例：
```
[s]
```
  
### <a id="wait"></a> **[wait] 暂停游戏**

该标签将暂停游戏一段设定好的时间，等待玩家的输入或自动继续。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|time|设定暂停的时间（单位为毫秒）||〇|
代码结构及示例：
```
[wait time="1000"]
```
  
### <a id="wait_cancel"></a> **[wait_cancel] 取消暂停**

该标签用于取消之前设置的暂停效果，使游戏继续进行。前提是之前已经使用了[wait]标签设置了暂停效果，并且暂停尚未结束。

这种标签适用于在“等待”过程中通过点击按钮或其他方式强行跳过当前场景，然后在跳转目的地处取消等待的情况。

代码结构及示例：
```
[wait_cancel]
```
  
### <a id="close"></a> **[close] 关闭窗口**

顾名思义，对于PC来说，就是关闭了游戏窗口；对于浏览器来说，就是关闭了标签页（选项卡）。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|ask|设定在关闭窗口之前是否显示确认对话框。值为“true”时会显示确认对话框，值为“false”时则直接关闭窗口|true|

代码结构及示例：
```
[close ask="true"]
```
  
### <a id="savesnap"></a> **[savesnap] 创建保存快照**

暂时保存当前的游戏状态。之后，通过使用[setsave]标签，此处记录的保存数据将会被存储起来。

该标签似乎就像将自动保存的过程进行了拆分。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|title|设定保存快照的标题。该标题将显示在保存界面中，帮助玩家识别不同的保存点||〇|

代码结构及示例：
```
[savesnap]
```
  
### <a id="autosave"></a> **[autosave] 自动保存**

当引擎读取到此标签后，便会自动保存当前游戏状态。并且设置sf.system.autosave为true。

通过在标题画面之前编写判断逻辑，您就可以创建一个机制，以便从上次保存的状态重新开始游戏。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|title|设定自动保存的存档标题。该标题将显示在保存界面中，帮助玩家识别不同的保存点|

代码结构及示例：
```
[autosave]
```
一个简单的判断机制：
```
[if exp="sf.system.autosave === true"]
 自动保存数据已经存在，是否读取它？[r]
  [link target=select1]【1】是[endlink][r]
  [link target=select2]【2】否[endlink][r]
  [s]

  *select1
  加载中...[l]
  [autoload]

  *select2
  [cm]
  不读取[l][r]

[else]

  没有自动保存的数据[l][r]

[endif]

自动保存开始[autosave][l][r]
请重新开始游戏[s]
```

  
### <a id="autoload"></a> **[autoload] 自动读取**

该标签用于自动读取之前通过[autosave]标签保存的游戏状态。当引擎读取到此标签时，它将尝试加载最近一次自动保存的数据，并将游戏状态恢复到那个保存点。

代码结构及示例：
```
[autoload]
```

### <a id="checkpoint"></a> **[checkpoint] 注册检查点**

您可以通过使用[rollback]标签返回到检查点位置。该标签可以用于快速跳转到选项位置，或者在调试过程中快速跳转到某个位置。

创建过多的检查点可能会导致游戏运行速度变慢。请将检查点的数量控制在最少，并使用[clear_checkpoint]标签删除不必要的检查点。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|name|设定检查点的名称。该名称可用于识别不同的检查点||〇|

代码结构及示例：
```
[checkpoint name="example_checkpoint"]
```

### <a id="rollback"></a> **[rollback] 回滚**

该标签用于回滚到之前通过[checkpoint]标签注册的检查点位置。当引擎读取到此标签时，它将尝试将游戏状态回滚到指定检查点的位置。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|checkpoint|设定要回滚到的检查点的名称||〇|
|variable_over|设定回滚时是否保留当前的游戏变量|true|
|bgm_over|设定回滚时是否保留当前的背景音乐|false|

代码结构及示例：
```
[rollback checkpoint="example_checkpoint" variable_over="true" bgm_over="false"]
```

### <a id="clear_checkpoint"></a> **[clear_checkpoint] 删除检查点**

该标签用于删除之前通过[checkpoint]标签注册的检查点。当引擎读取到此标签时，它将删除检查点，使其不再可用。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|name|设定要删除的检查点的名称。如果该值为空，则将清除全部的检查点|

代码结构及示例：
```
[clear_checkpoint name="example_checkpoint"]
```

### <a id="screen_full"></a> **[screen_full] 游戏画面全屏幕**

该标签将会使游戏窗口全屏。再次使用该标签将回退到窗口模式。

该标签只适用于PC平台。

代码结构及示例：
```
[screen_full]
```
  
### <a id="dialog"></a> **[dialog] 显示确认对话框**

该标签用于在游戏中显示一个确认对话框。当引擎读取到此标签时，它将显示一个对话框，玩家可以选择“是”或“否”。根据玩家的选择，游戏将继续进行或执行其他操作。

对话框类型为：警告框（alert）、确认框（confirm）、输入框（input）

※ 输入框已过时。不建议使用。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|type|设定对话框类型。请用上面的对话框类型来设置该属性|alert|
|name|该属性用于输入对话框，设定用于存储输入值的变量名称|tf.dialog_value|
|text|确认对话框里要显示的文本|
|storage|设定点击确定后跳转的剧本文件。若未指定，则将使用当前的剧本文件|
|target|设定点击确定后跳转的标签。若未指定，则使用当前剧本文件的开头标签。**如果同时省略了“storage”和“target”这两个参数，那么该脚本将直接跳转到下一个标签**|
|storage_cancel|设定点击取消后跳转的剧本文件。若未指定，则将使用当前的剧本文件|
|target_cancel|设定点击取消后跳转的标签。若未指定，则使用当前剧本文件的开头标签|
|label_ok|设定确定按钮的文本|OK|
|label_cancel|设定取消按钮的文本|Cansel|

代码结构及示例：
```
[dialog type="confirm" text="你确定要继续吗？" storage="example.ks" target="*example_label" storage_cancel="example.ks" target_cancel="*cancel_label" label_ok="是" label_cancel="否"]
```

  
### <a id="apply_local_patch"></a> **[apply_local_patch]（后期更新维护用）加载补丁文件**

该标签的版本适用范围是V470及以上，仅在作为PC应用程序发布时有效。

此标签会在到达指定位置时将补丁文件应用到游戏中。当更新是data文件夹之外的文件（如tyrano主文件）时，请在启动时使用更新过程，而非使用此标签。

如果补丁文件较大，游戏可能会暂时停止运行。最好显示诸如“正在加载......”这样的文本来告知玩家。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|file|设定要加载的补丁文件。该文件路径要以可执行文件（.exe）所在的文件夹作为起点||〇|
|reload|设定加载补丁文件后是否自动重新启动游戏|false|

代码结构及示例：
```
[apply_local_patch file="test.tpatch" reload="true"]
```
  
### <a id="check_web_patch"></a> **[check_web_patch]（后期更新维护用）检查更新补丁**

该标签的版本适用范围是V470及以上，仅在作为PC应用程序发布时有效。

如果服务器上有更新补丁，系统将会自动弹出提示信息，提示用户安装该补丁（安装补丁后需重新启动游戏，补丁才能生效）。

这需要租用一台服务器，并将json文件和tpatch文件上传到该服务器上。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|url|设定补丁 json 文件的 URL。||〇|

代码结构及示例：
```
[check_web_patch url="http://example.com/patch.json"]
```
  
### <a id="closeconfirm_on"></a> **[closeconfirm_on] 开启未保存确认结束游戏效果**

该标签启用后，将会设置游戏此时为“未保存”状态，当玩家尝试关闭游戏窗口时，系统将会弹出一个确认对话框，询问玩家是否要结束游戏。

当玩家保存或加载游戏时，游戏会变为“已保存”状态。此时再结束游戏将不会再弹出确认对话框。

代码结构及示例：
```
[closeconfirm_on]
```
  
### <a id="closeconfirm_off"></a> **[closeconfirm_off] 关闭未保存确认结束游戏效果**

该标签启用后，当玩家尝试关闭游戏窗口时，系统将不会弹出一个确认对话框，询问玩家是否要结束游戏。

代码结构及示例：
```
[closeconfirm_off]
```
