### <a id="clearstack"></a> **[clearstack] 清除栈**

清空由系统管理的栈。建议将此标签放置在没有需要返回的栈的场景中（例如在标题或章节的开头，或者其他自然的分段处）。

栈是一个用于存储用于通过诸如[call]、[if]或[macro]等标签传递时返回到调用源的信息的内存区域。（可以将其想象成在探索森林时留下的指路标，帮助你找到返回的路径。）

通常，在使用[return]、[endif]、[endmacro]后会清空栈。然而，如果在使用[call]、[if]、[macro]中反复跳转的情况下跳转到不同的场景，未收集的栈就会累积起来。

如果未收集的栈积累过多，可能会导致游戏运行速度变慢，或者使存档文件变得过大。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|stack|可以指定“call”、“if”和“macro”参数。这将能够清空特定的栈。如果不指定，则会清空所有栈。V515版本及以上：还可以指定“anim”参数。当指定“anim”时，它会将当前的动画数量重置为零，并确保下一个标签使用[wa]进行处理。如果省略了栈参数，则不会执行此操作|

代码结构及示例：
```
[clearstack stack="call"]
```
  
### <a id="if"></a> **[if] 条件分支**

该标签与JavaScript中的if语句类似，用于根据条件执行不同的代码。

一些常见的关系：  
A == B 等于  
A === B 全等  
A !== B 不全等  
A > B 大于  
A >= B 大于等于  
A < B 小于  
A <= B 小于等于

该标签与[elsif]、[else]和[endif]标签一起使用。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|exp|设定条件表达式，该表达式必须是一个JavaScript表达式||〇|

代码结构及示例：
```
[if exp="condition"]
example code 1
[elsif exp="condition2"]
example code 2
[else]
example code 3
[endif]
```
  
### <a id="elsif"></a> **[elsif] 复数调节分支**

该标签与JavaScript中的else_if语句类似，当[if]条件不满足时，可以使用该标签设定另一个条件分支。

但要注意该标签必须在[if]标签和[else]标签之间使用，并且只能与[if]标签和[else]标签一起使用。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|exp|设定条件表达式，该表达式必须是一个JavaScript表达式||〇|

代码结构及示例：
```
[elsif exp="condition2"]
```
  
### <a id="else"></a> **[else] 不满足时的条件分支**

该标签与JavaScript中的else语句类似，当[if]条件和[elsif]条件都不满足时，可以使用该标签设定一个默认的条件分支。

代码结构及示例：
```
[else]
```
  
### <a id="endif"></a> **[endif] 结束条件分支**

该标签用于结束条件语句，并且必须与[if]标签一起使用。该标签的目的是，将[if]和[endif]构成一个完整的条件语句块，从而避免对剧本的其他内容产生影响。

代码结构及示例：
```
[endif]
```
  
### <a id="call"></a> **[call] 启动子程序**

该标签用于引入外部剧本文件或跳转标记。

这个标签经常用于一个典型的场景：当你需要同时定义多个角色时，便可以将每个角色的定义放在一个单独的剧本文件中，然后在主剧本中使用[call]标签来引入这些角色定义文件。

以及这种典型场景：需要跳转到一个外部文件的特定内容，但是需要返回跳转前的文件里。因此该标签经常与[return]标签一起使用。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|storage|设定要引入的剧本文件。如果未指定，则跳转到当前文件|
|target|设定要跳转到的标记。如果未指定，则跳转到当前文件的开头|

代码结构及示例：
```
[call storage="subroutine.ks" target="*label1"]
```
  
### <a id="return"></a> **[return] 离开子程序**

该标签用于返回[call]标签的位置，并返回该地方后面继续执行剧本。

代码结构及示例：
```
[return]
```
  
### <a id="macro"></a> **[macro] 自定义标签**

该标签用于定义一个自定义标签。该标签必须与[endmacro]标签一起使用。

该标签的应用场景是，需要构建一个经常用到的重复性功能，或一个特定的功能，但又不想每次都写一大段代码来实现它，那么就可以使用该标签来定义一个自定义标签，然后在需要使用这个功能的地方直接使用这个自定义标签即可。

如果在参数值的开头加上一个“%”，则可以重复使用传递给宏的值。您还可以使用“|”来为属性指定默认值。

此外，如果在自定义标签里的标签写入“*”，那么该标签的所有属性将传递到自定义标签里，并且可以在自定义标签里使用。

tips: 如果你想在自定义标签里定义一个专属的属性，那么你应该在自定义标签里使用mp.your_var，这样your_var就变成了一个标签的属性可以赋值。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|name|设定自定义标签的名称。注意，不可以使用TyranoScirpt内部标签，text和label也不可以||〇|

代码结构及示例：
```
[macro name="customtag" your_var="example" your_var_2="example_1"]
[eval exp="mp.your_var=%your_var_2|example_1"]
[if exp="mp.your_var==example"]
[font *]example
[endif]
[endmacro]
```
  
### <a id="endmacro"></a> **[endmacro] 结束自定义标签**

该标签用于结束自定义标签模块，并且必须与[macro]标签一起使用。该标签的目的是，将[macro]和[endmacro]构成一个完整的自定义标签模块，从而避免对剧本的其他内容产生影响。

代码结构及示例：
```
[endmacro]
```

该标签用于清除自定义功能。

这个标签的使用场景一般如下：  
你设计了一个自定义标签，却发现在后期它会造成影响而不能用，但是在前期你还不能删除它。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|name|要删除的自定义标签的名字||〇|

代码结构及示例：
```
[erasemacro name="customtag"]
```
  
### <a id="ignore"></a> **[ignore] 在某条件下跳过剧本**

当满足该标签设定的条件时，将跳过由[ignore]和[endignore]囊括的部分，将不再执行代码和显示文本。该标签的作用类似于注释。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|exp|设定条件表达式||〇|

代码结构及示例：
```
[ignore exp="f.name=examle"]
example text
[endignore]
```

### <a id="sleepgame"></a> **[sleepgame] 暂停游戏**

当引擎读取到此标签时，引擎会临时保存游戏当前的状态，此时可以跳转到另一个场景。

此标签的一个常见使用场景是，在游戏进程过程中打开设置界面进行配置设置。先使用[sleepgame]来保存进度，然后跳转到配置场景文件，之后再用[awakegame]标签返回游戏。

[sleepgame]标签不能多次执行。换句话说，在已暂停状态下，不能再次调用[sleepgame]标签。请确保跳转目的地始终存在 [awakegame]标签。如果在暂停状态下碰到[breakgame]标签，暂停状态将被丢弃且不会返回。

还可以通过在[button]标签中指定属性role=sleepgame来注册[sleepgame]的行为。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|storage|设定点击确定后跳转的剧本文件。若未指定，则将使用当前的剧本文件|
|target|设定点击确定后跳转的标签。若未指定，则使用当前剧本文件的开头标签
|next|设定是否通过[awakegame]标签返回时自动跳转到下一个标签。该属性设置的意义是，当[awakegame]标签执行后，无需玩家点击，引擎会自动向下执行代码，直到遇到标签|true|

代码结构及示例：
```
[sleepgame storage="example.ks" target="*example" next="true" ]
```
  
### <a id="awakegame"></a> **[awakegame] 继续游戏**

当引擎读取此标签时，游戏状态将恢复到执行[sleepgame]标签时的状态。此标签在您希望在游戏中途暂时切换屏幕时非常有用。

就像加载保存的数据时一样，当游戏恢复时，会检查make.ks文件。如果您想对暂停期间执行的操作进行特殊处理，可以在make.ks中使用游戏变量来确定游戏是不是从[awakegame]恢复的。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|variable_over|设定恢复状态后是否继承当前的变量|true|
|bgm_over|设定恢复游戏状态后是否保持当前播放的bgm|true|

代码结构及示例：
```
[awakegame variable_over="true" bgm_over="true" ]
```
  
### <a id="breakgame"></a> **[breakgame] 清除存储的游戏暂停数据**

该标签用于清除[sleepgame]存储的游戏暂停的数据。

建议在不再需要返回暂停状态时使用“[breakgame]”功能。

代码结构及示例：
```
[breakgame]
```
