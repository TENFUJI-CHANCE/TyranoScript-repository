### <a id="glyph"></a> **[glyph] 设置等待点击图象的图形**

该标签用于自定义点击等待图标（在等待点击时显示在消息末尾的图像，默认是跳动的小点）。该标签可以更改所使用的图像，或者将其显示在屏幕上的固定位置，而非消息末尾。

点击等待图标可用以下类型的内容：
1. 图像文件（使用line属性定义）——GIF 和 WebP 格式也支持
2. 形状（使用figure属性）
3. 帧动画（使用koma_anim属性）
4. 直接指定的HTML（使用html属性）——供擅长HTML的作者使用

除了图像，还可以从预设形状中选择点击等待图标。您可以自由指定形状的颜色。

也可以选择并应用预设动画，或者使用[keyframe]标签定义的自定义关键帧动画。请注意，每次使用[glyph]标签时，点击等待图标的设置都会重置（包括未指定的属性）。请注意，您不能在多个标签（如 [position] 或 [font]）之间定义设置。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|line|设定图像文件。默认情况下，会在tyrano/images/system文件夹中搜索图像（其中 nextpage.gif文件位于该文件夹内）。可以通过“folder”参数更改路径|nextpage.gif|
|fix|设定是否固定显示位置而非消息末尾，若要固定，请设定为“true”|false|
|left|设定点击图像的x轴位置，该属性需要fix=true|0|
|top|设定点击图像的y轴位置，该属性需要fix=true|0|
|folder|设定点击图像文件所在的文件夹|tyrano/images/system|
|width|设定点击图像的宽度|
|height|设定点击图像的高度|
|marginl|设定点击图像的左外边距，该属性应用于确定点击图像与文本的相对位置|3|
|marginb|设定点击图像的下外边距，该属性应用于确定点击图像与文本的相对位置|0|
|anim|设定应用于点击图像的动画效果。<br />可使用的关键字如下：`flash_momentary`（瞬间闪光）、`flash`（平滑闪光）、`spin_x`（围绕 X 轴旋转）、`spin_y`（围绕 Y 轴旋转）、`spin_z`（围绕 Z 轴旋转）、`bounce`（弹跳）、`rotate_bounce`（旋转时弹跳）、`soft_bounce`（软弹跳）、`zoom`（缩放）|
|time|设定动画的持续时间|
|figure|设定点击图像为形状。<br />可用的关键字如下：`circle`（圆圈）、`triangle`（三角形）、`v_triangle`（向下三角形）、`rectangle`（长方形）、`diamond`（菱形）、`start`（星形）|
|color|设定使用形状作为点击图像时的颜色|0xffffff|
|name|设定用于点击图像的类的名字|
|html|设定点击图像的HTML内容，可以使用svg|
|keyframe|设定要应用于点击图像的关键帧动画（name）。不能与“anim”一起使用|
|easing|设定动画的缓动。<br />可用关键字如下：`ease`（起始和结束时播放效果平滑）、`linear`（恒定速度播放）、`ease-in`（开始时缓慢）、`ease-out`（结束时缓慢）、`ease-in-out`（开始时缓慢，结束时也缓慢）、`cubicbezier` （自定义缓动函数）|
|count|设定动画重复的次数。`infinite`表示无限循环|
|delay|设定动画开始前的延迟等待时间。默认值为0，表示无延迟|0|
|direction|设定是否在偶数次循环时动画播放反转。使用`alternate`则表示在每一轮的第二次循环时进行反向播放|
|mode|设定在动画结束之后是否保留上一帧的状态（位置、旋转等）。设定为`forwards`来保留该状态，`none`则会恢复到动画开始前的状态|forwards|
|koma_anim|设定帧动画（keyframe）所使用的图像。该图像必须包含所有帧以水平排列的形式存于一个图像文件中|
|koma_count|设定帧动画中的帧数。若设定此参数，则可省略“koma_width”这一项|
|koma_width|设定帧动画中每一帧的宽度。若设定此参数，则可省略“koma_count”这一项|
|koma_anim_time|设定一个完整（周期）的帧动画的持续时间（单位：毫秒）|1000|

代码结构及示例：
```
[glyph]
Default click-wait glyph (GIF animation)[p]

[glyph fix="true" left="640" top="600"]
Fixed position at the bottom center of the screen[p]

[glyph figure="rectangle" anim="bounce" width="5" color="0xCEE7F5" marginl="15"]
Recreate the default click-wait glyph using a shape and animation[p]

[glyph figure="v_triangle" anim="flash_momentary" delay="200"]
A downward triangle that flashes momentarily[p]

[glyph figure="diamond" anim="flash"]
A diamond that flashes smoothly[p]

[glyph figure="circle" anim="soft_bounce" marginl="10"]
A circle that bounces softly[p]

[glyph figure="rectangle" anim="rotate_bounce"]
A rectangle that bounces while rotating[p]

[glyph figure="star" anim="spin_y" color="yellow"]
A star spinning around the Y-axis[p]

[glyph figure="star" anim="spin_x" color="yellow"]
A star spinning around the X-axis[p]

[glyph figure="star" anim="spin_z" color="yellow"]
A star spinning around the Z-axis[p]

[glyph figure="star" anim="zoom" color="yellow"]
A star that zooms[p]

[glyph koma_anim="stepanim.jpg" koma_count="4" width="28"]
Frame animation[p]

[glyph html=`<span style="color: white; font-size: 20px;">🥺</span>`]
Directly specified HTML[p]
```
  
### <a id="glyph_skip"></a> **[glyph_skip] 设置跳过模式的图形**

该标签用于设置在跳过模式下显示的字符样式。也就是跳过模式的标识。

1. 已设定use属性
2. 已设定delete属性
3. 两者均未明确设定

在三种模式下，表现方式有所不同。如果设置了use参数，那么就可以将屏幕上已经显示的元素（例如[ptext]）视为跳过模式下的字形。如果设置了delete参数，则可以删除由[glyph_skip]标签设置的定义。

如果两者均未指定，则其属性与[glyph]标签相同，也就是说，此时应使用[glyph]标签的属性。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|use|该属性用于将屏幕上已显示的元素视为跳过模式的符号。[ptext]和[image]所设定的name用于在此处使用|
|delete|删除标签对跳过标识的定义。若要删除定义请设定为“true”。请注意，该标签不能与use和其他属性一起用|
|Others|[glyph]标签属性，在实际使用中无需定义此属性，直接使用[glyph]标签的属性即可。设定后将固定此标识的位置|


代码结构及示例：
```
; Set fixed skip mode glyph
[glyph_skip fix="true" left="10" top="10" figure="diamond" anim="flash"  color="orange" time="400" width="80"]

; Treating text displayed with [ptext] as skip glyph
[ptext     name="glyph_skip" layer="message0" x="20" y="10" text="SKIP!" edge="4px orange" size="30"]
[glyph_skip use="glyph_skip"]

[glyph_skip delete="true"]
```
  
### <a id="glyph_auto"></a> **[glyph_auto] 设置自动模式的图形**

该标签用于设置在自动播放模式下显示的字符样式。也就是自动播放模式的标识。

此标签的解释与[glyph_skip]相同，只需要将跳过换为自动，两者使用的是同一逻辑。只是会在一些细微的地方有所区别。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|fix|设定是否在屏幕上设置标识于一个固定位置。仅对于自动模式标识，固定标识和非固定标识均可设置|
|use|该属性用于将屏幕上已显示的元素视为自动模式的符号。[ptext]和[image]所设定的name用于在此处使用|
|delete|删除标签对自动标识的定义。若要删除定义请设定为“true”。请注意，该标签不能与use和其他属性一起用|
|Others|[glyph]标签属性，在实际使用中无需定义此属性，直接使用[glyph]标签的属性即可。请注意此处不能默认fix=true|

代码结构及示例：
```
;set fixed auto mode glyph
[glyph_auto fix="true" left="10" top="10" figure="star"    anim="spin_y" color="green" time="5000" width="80"]

; Regarding auto mode glyphs,
; You can set auto mode glyphs at the end of messages along with fixed glyphs
[glyph_auto html="⌛" anim="flash"]

; Treating text displayed with [ptext] as auto glyph
[ptext     name="glyph_auto" layer="message0" x="20" y="10" text="AUTO" edge="4px green" size="30"]
[glyph_auto use="glyph_auto"]

; Erase all of the above settings
[glyph_auto delete="true" fix="true"]
[glyph_auto delete="true" fix="false"]
```

### <a id="body"></a> **[body] 游戏画面外层的设置**

该标签用于自定义游戏屏幕的大小以及游戏屏幕内的黑色部分，黑色部分用于平衡分辨率。可以更改黑色部分的颜色、设置黑色部分的图像，并在游戏过程中调整游戏屏幕的大小。

注意：使用此标签所做的更改在加载存档数据时不会被恢复。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|bgimage|设定黑色部分背景的图片文件。该文件请放在bgimage文件夹中|
|bgrepeat|指定背景图像的显示模式。默认情况下，它会同时在水平和垂直方向上重复显示。可用如下关键字：
`repeat-x`（仅水平重复）、`repeat-y`（仅垂直重复）、`round`（拉伸以覆盖整个屏幕，同时保持比例）、`no-repeat`（不重复）|
|bgcolor|设定背景颜色，以OxRRGGBB格式。若设置了该值，则bgimage将被忽略|
|bgcover|设定是否要将背景图片拉伸以覆盖整个屏幕|false|
|scWidth|在游戏进行过程中，设定游戏屏幕的原始宽度。此标签适用于响应式设计。相当于Config.tjs中的scwidth。注意：“原始”在此处指的是在将游戏屏幕缩放以适应窗口大小之前的状态|
|scHeight|在游戏进行过程中，设定游戏屏幕的原始宽度。此标签适用于响应式设计。相当于Config.tjs中的scHeight。|

代码结构及示例：
```
[body bgimage="example.jpg" bgrepeat="no-repeat" bgcolor="0xffffff" bgcover="false" scWidth="1280" scHeight="720" ]
```

### <a id="cursor"></a> **[cursor] 改变鼠标光标图像**

该标签用于自定义鼠标光标。使用的图像文件应放在image文件夹里，支持的格式有gif、png、jpg。

该标签可以在游戏过程中随时改变鼠标光标。如果要设定一个默认的光标图像，请在config.tjs里的cursorDefault设定默认的图像。

要恢复为系统的默认光标样式，请输入“default”。

提示：鼠标指针图像的推荐尺寸为32x32像素或更小。可用作鼠标指针的图像最大尺寸因浏览器而异，但通常不超过128x128像素。

尺寸过大的图像不能用作鼠标指针。此外，如果设置的图像尺寸大于32x32像素，当鼠标移动到屏幕边缘时，指针可能会恢复为默认状态。

注意：当启用开发者工具时，鼠标指针自动隐藏的功能可能无法正常工作。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|storage|设定鼠标指针的图像文件。图像文件请放在“data/image”文件夹里。当该值为“default”时将恢复为默认鼠标指针|
|x|设定鼠标指针图像的x轴位置，这个x轴是相对于鼠标左上角的原点的|0|
|y|设定鼠标指针图像的y轴位置，这个y轴是相对于鼠标左上角的原点的|0|
|type|设定在鼠标悬停在默认按钮上时的光标样式。若要改变样式，请为该参数指定“nointer”|default|
|auto_hide|设定在一段时间内无操作后，光标是否会自动隐藏。“true”表示启用自动隐藏功能，“false”则保持光标始终可见。还可以指定以毫秒为单位的时间（例如 2000）来设定自动隐藏的持续时间|
|click_effect|设定是否启用点击效果动画。值为“true”或“false”|
|e_width|设定点击效果动画的宽度|
|e_opacity|设定点击效果动画的透明度（范围在0到255）|
|e_time|设定点击效果动画的持续时间|
|e_color|设定点击效果动画的颜色|
|e_blend|设定点击效果动画的混合模式。它支持与 [layermode] 标签的“mode”属性相同的关键词，例如“normal”或“overlay”等|
|e_scale|设定点击效果的缩放百分比。例如，设置为200将会使该效果扩大到其原始大小的200%|

代码结构及示例：
```
[cursor storage="example.jpg" x="1" y="1" type="nointer" auto_hide="true" click_effect="true" e_width="300" e_opacity="255" e_time="2000" e_color="0x000000" e_blend="overlay" e_scale="200" ]
```

### <a id="set_resizecall"></a> **[set_resizecall] 自适应游戏画面**

当玩家设备屏幕的纵横比发生变化时，可以触发一个场景（剧本）。

该标签的目的是实现响应式设计。让游戏能够根据手机的分辨率变化。

例如，在从竖屏模式切换到横屏模式时，您可以执行一个脚本来调整横屏模式下的坐标。相反，在从横屏模式切换到竖屏模式时，您可以执行一个脚本来调整竖屏模式下的坐标。通过实现这一点，您可以创建支持多种布局的游戏。请注意，调用的场景必须在最后执行[return]操作。

更多教程请参考[https://tyrano.jp/usage/advance/responsive](https://tyrano.jp/usage/advance/responsive)

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|storage|设定要跳转到的场景文件名。若未设定，则默认使用当前的场景文件。||〇|
|target|设定要跳转到的标记。若未设定，将会跳转到剧本的最开始|

代码结构及示例：
```
[set_resizecall storage="example.ks" target="*example" ]
```
  
### <a id="dialog_config"></a> **[dialog_config] 确认对话框图像设定**

确定对话框，经常用于关闭游戏时用于确认是否关闭。该对话框分为两个部分，显示询问问题的文本部分和按钮部分。

这类对话框一共有四个类型：[dialog_config]、[dialog_config_ok]、[dialog_config_ng]、[dialog_config_filter]。

注意：使用此标签所做的更改在加载已保存的游戏时不会被恢复。因此，在加载保存的游戏后在游戏过程中临时修改对话设计可能会存在bug。建议在进入标题画面之前（例如在 first.ks 文件中）在启动阶段就指定此标签。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|btntype|设定按钮类型。属性值请参考[glink]的color属性|
|btnwidth|设定按钮的宽度|
|btnmargin|设定按钮的外边距。支持以逗号分隔统一设定，例如 10,20 表示垂直边距为 10，水平边距为 20|
|btnpadding|设定按钮的内边距。支持以逗号分隔统一设定，例如 10,20 表示垂直边距为 10，水平边距为 20|
|fontsize|设定对话框内字体大小|
|fontbold|设定是否字体加粗，若要加粗请设定为“true”|
|fontface|设定文本字体类型|
|fontcolor|设定文本字体颜色|
|btnfontsize|设定按钮内的文本大小|
|btnfontbold|设定是否按钮内的文本字体加粗，若要加粗请设定为“true”
|btnfontface|设定按钮内的文本字体类型|
|btnfontcolor|设定按钮内的文本字体颜色|
|boxcolor|设定对话框的背景颜色|
|boxopacity|设定对话框的透明度，数值范围在0到255（255表示完全不透明）|
|boxradius|设定对话框的圆角角度|
|boxwidth|设定对话框宽度|

### <a id="dialog_config_filter"></a> **[dialog_config_filter] 确认对话框的滤镜背景设定**

此标签允许您设定滤镜，该条件在显示对话框时将适用于整个背景。可用的属性与[filter]标签中的属性相同。

|变量名|解释|默认值|必需|

### <a id="mode_effect"></a> **[mode_effect] 跳过（自动、暂停）模式变化效果**

该标签的作用是，在屏幕中显示一个图像（瞬时或永久），表示自动、跳过等模式。例如按住跳过键后，在屏幕中央会显示一个图像表示快进模式。

您可以在屏幕中央按照以下时间间隔来显示效果：
1. 当跳过模式启动时
2. 当自动模式启动时
3. 当跳过模式或自动模式停止时
4. 当保持跳过模式启动时
5. 当保持跳过模式停止时

※ 保持跳过指的是“仅在按住按钮或按键时才生效的跳过功能”。它通常被分配到键盘上的 Ctrl 键上。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|skip|跳过模式启动时所呈现的效果。可用的值为“none”、“default”或在image文件夹里的图像文件。none表示无效果，default表示默认效果，而指定一个图像文件则会显示该图像|
|auto|自动模式启动时所呈现的效果。其设定方式与跳过属性的设定方式相同|
|stop|当跳过模式和自动模式取消时所呈现的效果。其设定方式与跳过属性的设定方式相同|
|holdskip|当保持跳过模式启动时所呈现的效果。其设定方式与跳过属性的设定方式相同|
|holdstop|当保持跳过模式停止时所呈现的效果。其设定方式与跳过属性的设定方式相同|
|all|该属性可以统一设定上面五种属性。其设定方式与跳过属性的设定方式相同|
|env|设定效果的适用范围。范围限于移动设备和电脑，若设定“phone”，则该效果只在移动设备上有效；若设定“pc”，则该效果只在电脑上有效；若设定“all”，则在移动设备和电脑都有效|all|
|width|设定该效果的宽度|
|height|设定该效果的高度|
|color|设定图案部分的颜色（使用default效果才可用）|
|bgcolor|指定环绕形状的圆形部分的颜色（使用default效果才可用）|


代码结构及示例：
```
[mode_effect skip="none" auto="example.jpg" stop="default" holdskip="example.jpg" holdstop="none" all="default" env="all" width="100" height="100" color="0x000000" bgcolor="0xffffff" ]
```
  
### <a id="loading_log"></a> **[loading_log] 等待加载画面的设定**

在游戏暂时暂停（例如在加载资源或进行保存操作时）的情况下，可以在屏幕边缘显示一条“正在加载……”的提示信息。

属性：
|变量名|解释|默认值|必需|
|-|-|-|-|
|preload|设定资产加载时显示的文本内容。如果设定为“none”，将禁用显示。如果设定为“default”，则会显示默认的文本。如果设定为“notext”，则会显示加载图标但不显示任何文字|
|save|设定在保存时显示的文本内容。本属性的属性值与preload属性相同|
|all|同时设定预加载和保存属性。例如，指定“all='default'" 将为所有日志设置默认文本|
|dottime|设定动画中“...”符号逐个出现的时间（以毫秒为单位）。如果设定为“0”，则会禁用这些点的动画效果|
|icon|设定是否显示加载图标，使用“true”和“false”设定。如果要隐藏加载图标，只显示文字，则应设定为“false”|

代码结构及示例：
```
[loading_log preload="none" save="default" all="notext" dottime="1000" icon="false" ]
```
